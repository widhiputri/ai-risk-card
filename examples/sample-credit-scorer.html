Purpose and Usage

Intended Uses, Users and Domains
Predicts the probability of default (PD) for retail loan applicants. Intended for use by credit analysts and automated decisioning systems within retail banking. The model outputs a risk score (0-1000) and a risk tier (Low / Medium / High) per applicant.

Not intended for use in SME or corporate lending.
Usage Guidance
The model score should be used as one input within a broader credit assessment process. Final credit decisions must include human review for any applicant scored in the Medium or High tier.

Maximum recommended batch size: 500 applicants per run. Model performance is validated for Singapore-resident applicants only.

Output interpretation: scores above 750 indicate low default probability. Scores below 400 require mandatory human escalation.
TD

Techniques and Development

Main AI Techniques
Gradient boosting ensemble (XGBoost) trained on 5 years of historical retail loan performance data. Features include income, debt-to-income ratio, employment tenure, credit bureau tradeline data, and repayment history.
Architecture and Components
Single-model architecture. No generative or neural network components. Model is serialised as a binary artifact and served via a REST API. Inputs are validated and normalised by a pre-processing pipeline before scoring.
External AI Service Calls
Credit bureau data is retrieved at inference time via a real-time API call to CBS (Credit Bureau Singapore). No other external AI services are called.

Risks

5
▶ Insufficient model accuracy / soundness Robustness & Stability
Mitigation Strategy
The model is evaluated quarterly on a holdout dataset of recent loan originations. Performance thresholds (Gini > 0.45, KS > 0.30) are enforced before each production deployment. A champion-challenger framework is in place to compare new versions against the production model before promotion.
Appropriate Use Guidance
Do not use model scores for applicants outside the validated population (non-residents, SME borrowers). Monitor the Gini coefficient monthly; escalate if it drops below 0.40.
▶ Lack of explainability Transparency
Mitigation Strategy
SHAP (SHapley Additive exPlanations) values are computed for each applicant at inference time and logged. The top 3 contributing features and their directional impact are available to credit analysts via the decisioning portal.
Appropriate Use Guidance
For adverse action notices, use the SHAP-derived top features to generate compliant explanations to customers. Do not quote the raw score to customers without context.
▶ Unfair outcomes for protected groups Fairness & Bias
Mitigation Strategy
The model is evaluated against gender and age cohorts using Equal Opportunity Difference (EOD) and Statistical Parity Difference (SPD) metrics at each quarterly review. Features that are direct proxies for protected attributes (e.g. postal district as a proxy for race) have been excluded from the feature set.
Appropriate Use Guidance
If EOD or SPD exceeds 0.05 in any quarterly review, a fairness remediation workstream must be initiated before the next model deployment.
▶ Breach or misalignment with regulatory or organisational standards Legal & Regulatory
Mitigation Strategy
The model was reviewed by the Legal and Compliance team prior to deployment. An adverse action notice process compliant with MAS Notice 632 is in place. Model outputs are logged and retained for 5 years for regulatory audit purposes.
▶ Inadequate human oversight Accountability & Governance
Mitigation Strategy
All Medium and High risk tier decisions require credit analyst review before a decision is communicated to the applicant. A daily report flags any automated approvals in the Low tier for spot-check review. An override log tracks analyst departures from model recommendations.

Retrieves tradeline data from Credit Bureau Singapore at inference time.
Data Flow Description
Applicant data is submitted by the FI's loan origination system to the scoring API endpoint (POST /v1/score).

The pre-processing pipeline validates and normalises inputs, then calls the CBS API client to retrieve tradeline data.

The combined feature vector is passed to the XGBoost model, which outputs a score and risk tier.

The SHAP explainer computes feature attributions on the same input.

The API response returns: score (integer 0-1000), tier (Low/Medium/High), top 3 SHAP features, request ID, and timestamp.

All request/response pairs are written to the FI's audit log. No data is retained by FinAI Solutions infrastructure.
